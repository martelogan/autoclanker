# Excalidraw Reconciliation

Return a short change plan before editing the registry:

- `accepted_changes`: durable registry changes inferred from the board.
- `ignored_visual_changes`: colors, freehand notes, spacing, or decorative edits
  that will not survive regeneration.
- `ambiguous_changes`: anything that needs confirmation.
- `validation`: commands run and their result.

Matching heuristics:

- Prefer explicit big-bet IDs, issue numbers, and slugs when visible.
- Excalidraw element IDs generated by `bigbets` are stable, but collaborative
  tools may clone or rewrite them; do not rely only on element IDs.
- Position changes only affect layer/order. They never create dependency edges.
- Edge changes only count when a visible arrow connects two matched big-bet
  nodes and the direction is clear.
- Text-only issue references can move an idea family only when exactly one
  existing registry row matches the issue number or slug.

Do not write host-specific persistence, tracker APIs, or private deployment
logic into generic `bigbets` while reconciling a board.
